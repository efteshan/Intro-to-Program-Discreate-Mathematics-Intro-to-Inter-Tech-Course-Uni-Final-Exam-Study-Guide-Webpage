Part A: Lists
Mutable, ordered collection · Problems 1–5
📌 Key Concept: What is a List? ▾
A list is an ordered, mutable (changeable) collection of items. You can add, remove, or change elements after creation. Lists use square brackets [ ].
Syntax & Common Operations
# Create a list
marks = [60, 70, 80]

# Add element
marks.append(90)       # [60, 70, 80, 90]

# Access by index (starts at 0)
print(marks[0])        # 60
print(marks[-1])       # 90 (last item)

# Useful built-in functions
sum(marks)             # total
len(marks)             # count
max(marks)  min(marks)  # largest/smallest
OperationCodeResult
Add itemmarks.append(95)adds 95 to end
Remove itemmarks.remove(70)removes first 70
Lengthlen(marks)number of items
Sortmarks.sort()sorts in-place ascending
P1 Create a list of 5 numbers entered by the user and print the list ▾
💡 Concept
We create an empty list first, then use a for loop to ask the user 5 times. Each entered number is added to the list using .append().
✅ Solution
nums = []  # empty list
for i in range(5):
    n = int(input("Enter number: "))
    nums.append(n)
print("The list is:", nums)
Sample Output
The list is: [10, 20, 30, 40, 50]
🔍 How it works
  • nums = [] creates an empty list to hold values
  • range(5) repeats the loop exactly 5 times (i = 0,1,2,3,4)
  • int(input(...)) reads user input as a string, converts to integer
  • .append(n) adds each number to the end of the list

Part B: Tuples
Immutable, ordered records · Problems 6–8
📌 Key Concept: What is a Tuple? ▾
A tuple is like a list but immutable, once created, you cannot change its elements. Use parentheses ( ). Perfect for fixed records like student ID, name, GPA.
Syntax & Access
# Create a tuple
student = ("S001", "Asha", 3.75)

# Access by index
print(student[0])   # "S001"
print(student[1])   # "Asha"

# Slicing
print(student[-2:]) # ("Asha", 3.75)

# This CAUSES AN ERROR (tuples are immutable):
# student[1] = "Rahim"  ← TypeError!
List vs Tuple: Use a list when data needs to change (e.g. marks being added). Use a tuple for fixed records that should not change (e.g. student ID + name).

P7 Tuple slicing: create a tuple of 5 cities, print first city and last two ▾
💡 Concept
Slicing extracts a portion of a tuple. tuple[-2:] means "from 2nd-last to end." Negative indexes count from the right: -1 = last, -2 = second-to-last.
✅ Solution
cities = ("Dhaka", "Chittagong", "Rajshahi", "Khulna", "Sylhet")
print("First city:", cities[0])
print("Last two cities:", cities[-2:])
Output
First city: Dhaka Last two cities: ('Khulna', 'Sylhet')
🔍 How slicing works
  • Index 0 → first item → "Dhaka"
  • Index -1 → last item → "Sylhet"
  • Slice -2: → last 2 items → ("Khulna", "Sylhet")
  • Slice 1:3 → index 1 up to (not including) 3 → ("Chittagong", "Rajshahi")

Part C: Dictionaries
Key-value mapping · Problems 9–13
📌 Key Concept: What is a Dictionary? ▾
A dictionary stores data as key: value pairs inside curly braces { }. Instead of using an index number like a list, you look up values using a meaningful key (like a name or ID). Perfect for fast lookup.
Syntax & Operations
# Create
student = {"name": "Asha", "marks": 85}

# Access by key
print(student["name"])      # "Asha"
print(student.get("age", "N/A"))  # safe access

# Add / update
student["grade"] = "A"     # add new key
student["marks"] = 90     # update existing

# Loop through
for key, value in student.items():
    print(key, ":", value)

Functions with Data Structures
Lab Handout Tasks: passing lists, tuples, dicts to functions
📌 Key Concept: Functions in Python ▾
A function is a reusable block of code with a name. You define it once with def, then call it many times. Functions can accept data (parameters) and send back results (return).
Function Anatomy
def function_name(parameter1, parameter2):
    # code goes here
    return result

# Calling the function
answer = function_name(value1, value2)
print(answer)
Why use functions? Avoid repeating the same code. Break a big problem into small, named, testable pieces. Makes your program easier to read and fix.
T1 Function with List: calculate sum and average of marks ▾
💡 Concept
Pass a list as a parameter to a function. The function can use sum() and len() on it and return multiple values as a tuple.
✅ Solution
def calculate_marks(marks_list):
    total = sum(marks_list)
    average = total / len(marks_list)
    return total, average  # returns 2 values

marks = [50, 60, 70, 80]
total, avg = calculate_marks(marks)
print("Total:", total)
print("Average:", avg)
Output
Total: 260 Average: 65.0
🔍 How it works
  • Define the function with one parameter: marks_list (a list)
  • Compute total and average inside the function
  • Return BOTH values, Python packs them as a tuple automatically
  • total, avg = calculate_marks(marks) unpacks the returned tuple

S1 Step-by-step: Student Result Management System ▾
💡 Design Overview
We combine all three data structures: Tuple for fixed student names, List for marks (can change), and Dictionary to map name → mark for fast lookup. Functions handle each task cleanly.
Step 1: Input Data
names = ("Rahim", "Karim", "Nila", "Sadia")  # tuple
marks = [65, 80, 45, 90]                   # list
Step 2: Create Dictionary
def create_student_dict(names, marks):
    student_dict = {}
    for i in range(len(names)):
        student_dict[names[i]] = marks[i]
    return student_dict

students = create_student_dict(names, marks)
print(students)
Output
{'Rahim': 65, 'Karim': 80, 'Nila': 45, 'Sadia': 90}
Step 3: Display & Statistics
def display_students(student_dict):
    for name, mark in student_dict.items():
        print(name, "->", mark)

def calculate_statistics(marks):
    return max(marks), min(marks), sum(marks)/len(marks)

def count_passed(marks):
    return sum(1 for m in marks if m >= 50)

display_students(students)
high, low, avg = calculate_statistics(marks)
print("Highest:", high)
print("Lowest:", low)
print("Average:", avg)
print("Passed:", count_passed(marks))
Expected Output
Rahim -> 65 Karim -> 80 Nila -> 45 Sadia -> 90 Highest: 90 Lowest: 45 Average: 70.0 Passed: 3

S2 Menu-based Student Manager: full interactive program ▾
💡 Concept
A menu-driven program uses a while True loop to keep running, showing a menu each time. The user picks a number, and an if/elif/else chain runs the right action. break exits the loop.
✅ Full Program
students = {}

while True:
    print("\n--- Student Manager ---")
    print("1. Add student")
    print("2. Add marks")
    print("3. Search student")
    print("4. Display all")
    print("5. Exit")
    choice = input("Enter choice: ").strip()

    if choice == "1":
        sid  = input("ID: ").strip()
        name = input("Name: ").strip()
        students[sid] = {"name": name, "marks": []}
        print("Added.")

    elif choice == "2":
        sid = input("Enter ID: ").strip()
        if sid in students:
            m = int(input("Enter mark: "))
            students[sid]["marks"].append(m)
            print("Mark added.")
        else:
            print("Student not found.")

    elif choice == "3":
        sid = input("Enter ID: ").strip()
        if sid in students:
            print(sid, students[sid])
        else:
            print("Not found.")

    elif choice == "4":
        for sid, info in students.items():
            marks = info["marks"]
            avg = sum(marks)/len(marks) if marks else 0
            print(sid, info["name"], "Marks:", marks, "Avg:", avg)

    elif choice == "5":
        print("Program ended.")
        break
    else:
        print("Invalid choice.")
🔍 How each part works
  • Loop runs forever until break is hit (option 5)
  • Creates a new entry in the students dict with empty marks list
  • Finds student by ID, appends mark to their list
  • Looks up student by ID, instant access with dict
  • Loops all students, computes avg on the fly
  • avg = ... if marks else 0 prevents division by zero if no marks yet

📋 Quick Reference: When to use List, Tuple, or Dictionary? ▾
Structure When to use Example
List [ ] Collection of items that can change. Order matters. marks = [80, 90, 75]
Tuple ( ) Fixed record. Data should NOT change. ("S001", "Asha", 3.75)
Dictionary { } Fast lookup by name/ID. Key-value pairs. {"name": "Asha", "gpa": 3.75}
Function def Repeated logic. Any task you do more than once. def calculate_avg(marks):
Common Methods Cheat Sheet
# LIST
lst.append(x)     # add to end
lst.remove(x)     # remove first x
lst.sort()         # sort ascending
len(lst)           # number of items

# TUPLE
tpl[0]             # access first item
tpl[-1]            # access last item
tpl[1:3]           # slice items 1 and 2

# DICTIONARY
d["key"]           # get value
d["key"] = val     # add / update
d.items()          # loop key+value pairs
d.get("k", "N/A")  # safe access
"key" in d         # check if key exists
